# 量化行情展示项目：一期建设计划

## 1. 现有数据格式

一期样例文件为 `000852perf (1).xlsx`，检查结果如下：

- 工作表：`000852perf`
- 数据量：1211 条日频行情
- 日期范围：2021-06-19 至 2026-06-17
- 标的：`000852`（中证1000指数）
- 共 15 列：
  - 日期、指数代码及中英文名称
  - Open、High、Low、Close
  - 涨跌、涨跌幅
  - 成交量、成交金额、样本数量
- 日期和大部分行情数值在 Excel 中以文本形式保存，导入时需显式转换。
- 数据无空值，日期整体升序，OHLC 关系正常。
- 异常记录：`2021-06-19` 为周六，且行情与 `2021-06-21` 完全一致。导入时应提示警告，由用户决定是否保留，不应静默删除。

## 2. 一期开发目标

一期交付一个可在浏览器本地运行的行情分析应用，形成“文件导入—数据校验—行情展示—指标计算”的完整使用闭环。

### 2.1 数据导入目标

- 用户可以选择并导入固定格式的本地 `.xlsx` 行情文件。
- 系统能够识别当前样例中的中英文混合表头。
- 日期、价格、涨跌幅、成交量和成交金额能够转换为统一类型。
- 导入时执行字段完整性、日期顺序、重复记录、OHLC 关系及异常交易日校验。
- 严重错误阻止导入；可继续处理的异常以警告形式展示。
- 导入结果包含文件名、标的、日期范围、有效行数、错误数和警告数。

### 2.2 行情图表目标

- 展示蜡烛图主图和成交量副图。
- 支持滚轮缩放、拖拽平移和图表尺寸自适应。
- 使用十字光标跟随鼠标，并同步主图与副图的时间位置。
- 展示鼠标当前位置对应的日期、Open、High、Low、Close、涨跌、涨跌幅、成交量及成交金额。
- 默认采用国内市场红涨绿跌的颜色规则，并为后续配置化预留接口。

### 2.3 金融指标目标

- 实现 SMA、EMA、BOLL、MACD、RSI、KDJ、ATR、CCI、WR、OBV 和成交量均线。
- 指标支持添加、隐藏、删除、修改参数和恢复默认参数。
- 趋势指标叠加在蜡烛图主图，其余指标按类型显示在独立副图。
- 指标计算与页面和图表组件解耦，使用无副作用的 TypeScript 纯函数。
- 数据不足的预热区返回空值，不使用零填充。
- 每项指标均有固定样例、边界条件和精度测试。

### 2.4 质量与性能目标

- 当前样例文件能够一次完成解析和展示。
- 1 万条日线数据可以在浏览器中正常导入、计算和渲染。
- 图表缩放、拖拽及十字光标交互保持流畅。
- 文件解析、数据校验、指标计算和图表渲染保持模块隔离。
- 核心导入规则和指标计算具有自动化测试。

### 2.5 一期范围边界

一期不包含：

- 策略编写及策略管理。
- 买卖信号和订单撮合。
- 手续费、滑点、仓位和账户模型。
- 收益统计及策略回测。
- 用户系统、权限和云端文件存储。
- 在线行情及实时推送。

## 3. 一期技术栈

一期采用纯前端架构，不建设后端服务。Excel 文件在浏览器本地解析，原始数据不上传服务器。

### 3.1 核心技术选型

| 层级 | 技术 | 主要用途 |
| --- | --- | --- |
| 开发语言 | TypeScript | 数据模型、导入、指标及界面逻辑 |
| 前端框架 | React | 页面与组件组织 |
| 构建工具 | Vite | 本地开发、构建和静态部署 |
| 金融图表 | TradingView Lightweight Charts | 蜡烛图、成交量、缩放和平移 |
| UI 组件 | Ant Design | 文件上传、表格、表单、提示及布局组件 |
| Excel 解析 | SheetJS/xlsx | 在浏览器中读取 `.xlsx` 文件 |
| 数据校验 | Zod | 表头、导入数据和指标参数校验 |
| 状态管理 | Zustand | 行情数据、导入状态、指标配置和图表状态 |
| 单元测试 | Vitest | 导入规则、数据转换和指标计算测试 |

具体依赖版本在初始化项目时锁定，并提交依赖锁文件。

### 3.2 一期运行架构

```text
本地 Excel 文件
    ↓
浏览器文件读取与 Excel 解析
    ↓
表头映射、类型转换和数据校验
    ↓
统一 Candle[] 行情模型
    ├──→ 指标计算模块
    └──→ 蜡烛图及成交量图表
              ↓
       十字光标与行情详情
```

### 3.3 模块划分

```text
src/
  features/
    import/       # 文件读取、表头映射、转换和校验
    chart/        # 蜡烛图、成交量、十字光标及副图同步
    indicators/   # 指标定义、参数、计算及展示适配
  models/         # Candle、导入结果、错误和指标类型
  stores/         # Zustand 状态
  workers/        # 后续 Web Worker 计算入口
  components/     # 通用界面组件
  utils/          # 日期、数字和格式化工具
```

### 3.4 计算与性能策略

- 一期指标默认在浏览器主线程计算。
- 指标模块不得依赖 React 或具体图表库。
- 指标输入和输出按原始行情时间轴对齐。
- 当数据规模或指标数量导致明显卡顿时，将同一计算接口迁移至 Web Worker。
- 图表只接收已经标准化的行情和指标序列，不负责业务数据清洗或指标计算。

### 3.5 部署方式

- 一期构建为静态资源，可通过本地静态服务器、Nginx 或静态托管平台部署。
- 无数据库、后端服务、Redis 或消息队列依赖。
- 默认不将用户导入的数据发送到任何网络服务。

## 4. 二期技术预留

二期进入策略回测后，建议增加 Python 后端，同时保持一期前端可独立运行：

| 层级 | 建议技术 | 主要用途 |
| --- | --- | --- |
| API 服务 | FastAPI | 行情、指标、策略和回测接口 |
| 计算层 | NumPy + Polars/Pandas | 向量化指标及回测计算 |
| 关系数据库 | PostgreSQL | 用户、策略、任务和标的元数据 |
| 行情文件 | Parquet | 批量历史行情存储 |
| 异步任务 | Redis + RQ/Celery | 长时间运行的回测任务 |

前后端通过 OpenAPI 定义接口，并为前端生成 TypeScript 类型。正式回测结果以后端计算为准；前端指标主要用于即时交互和图表预览。

## 5. 统一数据模型

导入后的行情统一转换为内部模型：

```ts
interface Candle {
  time: string;
  symbol: string;
  open: number;
  high: number;
  low: number;
  close: number;
  change?: number;
  changePercent?: number;
  volume?: number;
  turnover?: number;
  constituentCount?: number;
}
```

原始行号及导入警告应单独保存，以便错误定位，不混入指标计算模型。

## 6. 功能模块

### 6.1 文件导入

一期范围：

- 支持 `.xlsx` 文件。
- CSV 可作为兼容项，优先级低于 Excel。
- 自动识别样例中的中英文混合表头。
- 将 `YYYYMMDD` 日期转换为标准日期。
- 将文本行情转换为数值。
- 指数代码按字符串处理，保留前导零。
- 一期支持单工作表、单标的、日频行情。
- 导入后显示文件名、标的、日期范围、有效行数及警告数量。

校验规则：

- 必填字段完整。
- 日期格式合法、无重复且顺序正确。
- 行情字段可转换为有限数值。
- `High >= Open/Close`。
- `Low <= Open/Close`。
- 最高价不低于最低价。
- 价格、成交量和成交金额不得为负。
- 非交易日记录提示警告。
- 相邻日期行情完全重复时提示警告。
- 错误及警告包含 Excel 行号和明确原因。

验收标准：

- 样例文件能够一次成功导入。
- 1211 条数据均可被读取和标准化。
- `2021-06-19` 的异常能够得到明确提示。
- 无法继续处理的错误与可继续处理的警告有明确区分。

### 6.2 蜡烛图

一期功能：

- 展示蜡烛图主图。
- 展示成交量副图。
- 十字光标跟随鼠标移动。
- 主图与副图的时间位置同步。
- 悬浮信息展示日期、Open、High、Low、Close、涨跌、涨跌幅、成交量和成交金额。
- 支持滚轮缩放和拖拽平移。
- 自动适配价格范围。
- 默认采用国内市场习惯：红涨绿跌。
- 图表随窗口尺寸变化自适应。

验收标准：

- 鼠标指向的数据与当前蜡烛严格对应。
- 主图与副图的十字光标保持同步。
- 缩放、平移以及重新导入文件后图表状态正常。
- 大数据量下无明显交互卡顿。

### 6.3 金融指标

一期首批指标：

- 趋势：SMA、EMA、BOLL
- 动量：MACD、RSI、KDJ
- 波动：ATR、CCI、WR
- 量价：OBV、成交量均线

设计要求：

- 指标参数可编辑并可恢复默认值。
- SMA、EMA、BOLL 叠加在蜡烛图主图。
- MACD、RSI、KDJ、ATR 等使用独立副图。
- 指标支持显示、隐藏和删除。
- 指标计算与图表组件解耦。
- 数据不足的预热区返回空值，不使用零填充。
- 每项指标具有固定样例、边界条件和数值精度测试。

## 7. 页面布局

- 顶部：文件导入、标的信息、日期范围及数据状态。
- 左侧：指标选择及参数设置。
- 中部：蜡烛图、成交量和指标副图。
- 右侧或浮层：十字光标所指向的行情详情。
- 底部：导入结果、异常警告及错误明细。

具体布局在 UI 原型阶段确认。

## 8. 测试计划

### 8.1 导入测试

- 当前样例 Excel。
- 空文件及空工作表。
- 缺少必填列。
- 表头名称错误或存在空格。
- 数字包含逗号、空格或百分号。
- 重复日期及乱序日期。
- 周末或其他非交易日日期。
- 非法 OHLC 数据。
- 单元格为空、非数字或无限值。

### 8.2 图表测试

- 只有少量行情数据。
- 数万条行情数据。
- 连续上涨、下跌和平盘数据。
- 缩放和平移边界。
- 主图和副图光标同步。
- 浏览器窗口尺寸变化。

### 8.3 指标测试

- 默认参数结果。
- 参数最小值及极端值。
- 数据不足时的预热区。
- 指标输出长度与原始行情对齐。
- 通过权威样例或第二套实现进行结果比对。

## 9. 性能目标

- 1 万条日线数据可在浏览器中正常导入和计算。
- 图表缩放、拖拽和十字光标保持流畅。
- 修改指标参数后能够及时更新。
- 文件解析、指标计算和图表渲染之间保持模块隔离，为 Web Worker 优化预留接口。

## 10. 实施顺序与工作量

预计约 10～12 个开发人日：

1. 工程骨架与统一数据模型：1 人日。
2. Excel 导入、字段映射与数据校验：2 人日。
3. 蜡烛图、成交量及十字光标：2～3 人日。
4. 指标引擎及图表叠加：3～4 人日。
5. 错误处理、性能优化和测试：2 人日。
