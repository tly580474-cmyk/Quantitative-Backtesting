# 因子研究使用指南

本文档说明前端「因子研究」模块的日常使用流程，包括研究快照检查、单因子研究、多因子合成、报告解读和常见注意事项。

## 1. 进入页面

打开前端页面后，进入顶部导航里的「因子研究」。

本地开发环境常用地址：

- 前端：`http://127.0.0.1:5558/`
- 后端：`http://127.0.0.1:3001`

后端必须保持运行，否则因子目录、研究快照状态和研究任务接口都无法使用。

## 2. 先检查研究快照

进入「因子研究」页面后，页面顶部会显示「研究快照」状态条。

重点关注以下信息：

| 字段 | 含义 |
| --- | --- |
| 快照 | 当前 DuckDB/Parquet 研究快照的数据日期和行数 |
| MySQL | MySQL 权威库的数据日期和行数 |
| 缺失日期 | 当快照落后时，列出尚未同步进快照的具体交易日 |
| 状态 | 当前快照是否可用于研究 |

状态说明：

| 状态 | 含义 | 建议操作 |
| --- | --- | --- |
| 已追平 | 快照与 MySQL 权威库一致 | 可以直接运行因子研究 |
| 待更新 | MySQL 有新数据，快照尚未包含 | 点击「更新快照」 |
| 需复核 | 快照领先或与 MySQL 不一致 | 暂停研究，检查数据来源 |
| 不可用 | 尚无可用快照 | 点击「更新快照」或先检查后端配置 |

如果状态是「待更新」，点击右侧「更新快照」。

当前更新逻辑为日级追加优先：例如只缺 `2026-07-09`，系统只生成这一天的 Parquet 分区，不会重写 2000 年以来的历史数据。若检测到历史日期被修正，系统才会按受影响年份重建。

## 3. 因子库

左侧「因子库」展示当前已发布的因子。

每个因子包含：

- 因子名称
- 因子 ID
- 方向
- 预热天数
- 依赖字段

方向说明：

| 方向 | 含义 |
| --- | --- |
| 高值优先 | 因子值越高越看好 |
| 低值优先 | 因子值越低越看好 |
| 研究观察 | 不预设方向，仅用于观察 |

点击某个因子后，右侧运行配置会自动选中它。

常见内置因子示例：

- `momentum_20`：20 日动量
- `reversal_5`：5 日反转

## 4. 运行单因子研究

在「运行配置」区域选择「单因子」标签。

需要填写：

| 参数 | 说明 |
| --- | --- |
| 因子 | 要研究的因子 |
| 研究区间 | 研究开始日期和结束日期 |
| 持有期 | 用因子预测未来多少日收益，例如 5 日 |
| 分层数 | 按因子值分成多少组，例如 5 层 |
| 市场 | 可选沪市、深市、北交所 |
| 成交额下限 | 可选，用于过滤流动性不足的股票 |

填写完成后，点击右上角「运行研究」。

运行成功后，页面会生成报告，并保存一条运行记录。

## 5. 运行多因子合成

在「运行配置」区域切换到「多因子合成」标签。

需要填写：

| 参数 | 说明 |
| --- | --- |
| 因子组合 | 选择多个因子 |
| 研究区间 | 整体研究区间 |
| 验证区间起点 | 可选，用于训练/验证拆分 |
| 持有期 | 用合成因子预测未来多少日收益 |
| 分层数 | 按合成因子值分成多少组 |
| 权重方式 | 等权、IC 加权、RankIC 加权或手动权重 |
| 市场 | 可选沪市、深市、北交所 |
| 成交额下限 | 可选，用于过滤流动性不足的股票 |

权重方式说明：

| 权重方式 | 含义 |
| --- | --- |
| 等权 | 每个因子权重相同 |
| IC 加权 | 按训练期 IC 分配权重 |
| RankIC 加权 | 按训练期 RankIC 分配权重 |
| 手动权重 | 用户手动输入每个因子的权重 |

手动权重格式示例：

```text
momentum_20:2,reversal_5:-1
```

表示：

- `momentum_20` 权重为 `2`
- `reversal_5` 权重为 `-1`

填写完成后，点击「运行多因子」。

## 6. 单因子报告解读

报告区域会显示核心指标：

| 指标 | 含义 |
| --- | --- |
| 样本数 | 参与计算的股票样本数量 |
| 平均 IC | 因子值和未来收益的相关性 |
| Rank IC | 因子排序和未来收益排序的相关性 |
| ICIR | IC 的稳定性指标 |
| 多空差 | 最高层和最低层的未来收益差 |

一般解读方式：

- IC 长期为正，说明因子方向较有效。
- Rank IC 稳定为正，说明因子排序能力较好。
- ICIR 越高，说明因子越稳定。
- 多空差越明显，说明分层收益越有区分度。

报告下方还包含：

- IC 序列：观察因子每日有效性是否稳定。
- 分层收益：观察高低层是否呈现单调性。
- 日度表格：查看每个交易日的 IC、Rank IC 和样本数。

## 7. 多因子报告解读

多因子报告额外包含：

| 指标 | 含义 |
| --- | --- |
| 因子数 | 参与合成的因子数量 |
| 平均相关 | 因子之间的平均绝对相关性 |
| 验证 IC | 验证区间内的 IC |
| 验证多空 | 验证区间内的多空收益差 |

重点关注：

- 因子之间相关性不要过高，否则组合意义有限。
- 训练期好、验证期也好，结果才更可信。
- 如果验证期明显变差，可能存在过拟合。

相关性矩阵解读：

| 相关性 | 含义 |
| --- | --- |
| 高正相关 | 两个因子可能重复 |
| 高负相关 | 两个因子可能互相抵消 |
| 接近 0 | 更适合作为组合候选 |

## 8. 运行历史

右下「运行历史」会列出最近的研究任务。

任务状态包括：

- 等待中
- 运行中
- 完成
- 失败
- 已取消

可用操作：

| 操作 | 说明 |
| --- | --- |
| 查看 | 打开已完成报告 |
| 重试 | 重新运行失败或取消的任务 |
| 取消 | 取消等待中或运行中的任务 |

## 9. 推荐日常流程

每天数据更新后，建议按以下流程使用：

1. 打开「因子研究」页面。
2. 查看顶部研究快照状态。
3. 如果显示「待更新」，点击「更新快照」。
4. 等状态变成「已追平」。
5. 先跑一个短区间单因子测试。
6. 再跑长区间单因子。
7. 最后做多因子合成。
8. 保存并对比运行历史里的结果。

## 10. 注意事项

不要在快照状态不是「已追平」时运行正式研究。

如果状态是「待更新」，说明 MySQL 已经有新数据，但研究快照还没包含这些交易日。

如果状态是「需复核」，通常说明快照比 MySQL 领先，或行数、日期出现异常。这时不要盲目点击更新，应先检查 MySQL 数据和快照来源。

如果只是日常新增交易日，可以点击「更新快照」。系统会按缺失日期追加，不会每天全量重写硬盘。

## 11. 常用命令

在后端目录执行：

```bash
cd server
```

检查快照状态：

```bash
npm run snapshot:freshness
```

日常更新快照：

```bash
npm run snapshot:build
npm run snapshot:verify
```

显式全量重建快照：

```bash
npm run snapshot:build -- --full
```

只重建指定年份：

```bash
npm run snapshot:build -- --years 2026
```

前端页面通常不需要手动执行这些命令；优先使用「因子研究」页面里的「更新快照」按钮。
