# 个股盘后增量更新运行手册

## 执行条件

- 仅处理 `type=stock AND status=active` 的证券；
- 仅允许交易日盘后 `final` 阶段执行，盘中手动触发会被拒绝，调度器也不会发起盘中个股增量更新；
- 盘后运行时写入 `is_final=1`；
- 无当日报价的停牌证券视为正常无数据，不写空 K 线；
- 已退市证券不会进入任务目标。

## 手动执行

```http
POST /api/sync/incremental
Content-Type: application/json

{}
```

接口返回 `jobId`。通过以下接口查看进度和证券级结果：

```text
GET /api/sync/jobs/:jobId
POST /api/sync/jobs/:jobId/retry
POST /api/sync/jobs/:jobId/cancel
```

正常流程：

1. 建立全部活跃证券的任务明细，立即显示任务总数；
2. 腾讯报价按 70 只分片、最多 6 路并发抓取；
3. 单个分片失败时保留其他成功分片，再按 25 只分片补取；
4. 日线、每日估值和旧表回滚副本分别按 500 行批量 upsert；
5. 主体写入完成后更新进度，只保留历史缺口和除权校验尾项；
6. 除权信号按发布因子重建规则校验，通过后原子发布新因子版本；
7. 失败证券保留错误码和错误信息，可重试。

## 字段口径

- 成交量统一保存为“股”，腾讯返回的“手”乘以 100；
- 成交额由腾讯“万元”转换为“元”；
- 换手率保存百分数值，例如 `0.55` 表示 `0.55%`；
- 总市值、流通市值由“亿元”转换为“元”；
- 总股本和流通股本由对应市值除以收盘价推导；
- PE(TTM)、PB、量比直接保存腾讯值；
- 腾讯当日报价不提供 PS(TTM)，因此按数据规范保留 `NULL`，不伪造估算值。
