# 全量历史行情导入操作手册

## 当前状态

5.5 阶段首个可运行切片已经支持：

- 读取“不复权”目录；
- 校验固定 37 列表头、代码、日期顺序和 OHLC；
- 将日线与每日估值拆分写入 v2 表；
- 文件 SHA-256、批次、文件状态和行数记录；
- 5 万行默认分片和文件级断点续传；
- 同一批次、同一文件校验和自动跳过；
- MySQL `LOCAL INFILE` 不可用时降级为批量 `REPLACE`；
- 非法 OHLC 原始行保留，并写入阻断级数据质量问题；
- `--dry-run` 只读预检。

复权参数迁移已经完成。Parquet/DuckDB 快照仍属于后续里程碑，当前不要执行
研究镜像的全市场正式发布。

## 配置

在 `server/.env` 中配置：

```dotenv
STOCK_HISTORY_ROOT=D:\github_public_repo\所有股票的历史数据\每只股票一个文件
```

也可以每次通过 `--source` 指定，不会把路径写入前端。

## 推荐执行顺序

### 1. 只读预检

```powershell
cd server
npm run import:history -- --limit 10 --dry-run
```

全目录预检：

```powershell
npm run import:history -- --dry-run
```

### 2. 指定小样本装载

```powershell
npm run import:history -- --codes 000001,000002,600000
```

或导入排序后的前 10 个文件：

```powershell
npm run import:history -- --limit 10
```

### 3. 断点续传

首次运行结束时会输出 `batchId`。使用同一批次继续：

```powershell
npm run import:history -- --batch-id <batchId> --limit 10
```

已完成且 SHA-256 未变化的文件会显示为 `skipped`。如果源快照已变化，导入器会拒绝
复用旧批次，避免把两个源版本混入同一批次。

## MySQL LOCAL INFILE

最快路径使用 `LOAD DATA LOCAL INFILE`。如果 MySQL 服务端未启用，导入器会自动降级为
每批 1,000 行的 `REPLACE`，功能不受影响，但全量导入会更慢。

正式全量演练前，应在受控环境评估并启用 `local_infile`。客户端只允许 MySQL 读取
导入器刚生成的指定临时分片，拒绝其他路径。

## 全量复权参数层

完成日期：2026-07-04。

执行命令：

```powershell
cd server
npm run import:factors -- --source <历史数据根目录>
```

腾讯历史文件包含现金分红形成的价格偏移，无法只用单一乘数精确表达。因此 v2 参数采用
仿射形式：

```text
前复权价格 = 不复权价格 × factor + price_offset
```

参数仍按公司行为变化区间压缩保存；`factor_version` 固定源快照，后复权文件仅用于
交叉验证，不作为第二份权威价格事实。源文件个别 OHLC 字段存在独立舍入或陈旧值时，
计算器保留可验证的多数价格并将偏差写入质量报告。

| --- | ---: |
| 前复权文件 | 5,825 |
| 完成文件 | 5,825 |
| 失败文件 | 0 |
| 有因子证券 | 5,824 |
| 压缩参数 | 58,421 |
| 平均每只证券参数 | 10.03 |

质量说明：

- `000508` 三个目录均只有表头，正常记录为零因子；
- 419 只证券的前复权重建通过率低于 99.5%，已写入 warning；
- 4,764 只证券的后复权交叉验证低于 99.5%，主要来自腾讯源历史/近期倍率不一致；
- 北交所缺少后复权文件不阻断前复权参数发布；
- 完整报告位于 `server/.cache/history-import/factor-report-<batchId>.json`。

## 安全约束

- 导入器不在 Web 服务启动时自动运行；
- 不复权是当前唯一权威价格事实层；
- 缺失值写入 `NULL`，不得用零替代；
- 当前只写 v2 影子表，不切换线上 Repository；
- 全量发布前必须核对 17,036,064 行基线；
- 临时 TSV 成功提交或失败回滚后都会删除。
