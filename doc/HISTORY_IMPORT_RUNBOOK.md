# 全量历史行情导入操作手册

## 当前状态

5.5 阶段首个可运行切片已经支持：

- 读取“不复权”目录；
- 校验固定 37 列表头、代码、日期顺序和 OHLC；
- 将日线与每日估值拆分写入 v2 表；
- 文件 SHA-256、批次、文件状态和行数记录；
- 5 万行默认分片和文件级断点续传；
- 同一批次、同一文件校验和自动跳过；
- MySQL `LOCAL INFILE` 不可用时降级为批量 `REPLACE`；
- 非法 OHLC 原始行保留，并写入阻断级数据质量问题；
- `--dry-run` 只读预检。

前复权、后复权的因子迁移和 Parquet/DuckDB 快照属于后续里程碑，当前不要执行
全市场正式发布。

## 配置

在 `server/.env` 中配置：

```dotenv
STOCK_HISTORY_ROOT=D:\github_public_repo\所有股票的历史数据\每只股票一个文件
```

也可以每次通过 `--source` 指定，不会把路径写入前端。

## 推荐执行顺序

### 1. 只读预检

```powershell
cd server
npm run import:history -- --limit 10 --dry-run
```

全目录预检：

```powershell
npm run import:history -- --dry-run
```

### 2. 指定小样本装载

```powershell
npm run import:history -- --codes 000001,000002,600000
```

或导入排序后的前 10 个文件：

```powershell
npm run import:history -- --limit 10
```

### 3. 断点续传

首次运行结束时会输出 `batchId`。使用同一批次继续：

```powershell
npm run import:history -- --batch-id <batchId> --limit 10
```

已完成且 SHA-256 未变化的文件会显示为 `skipped`。如果源快照已变化，导入器会拒绝
复用旧批次，避免把两个源版本混入同一批次。

## MySQL LOCAL INFILE

最快路径使用 `LOAD DATA LOCAL INFILE`。如果 MySQL 服务端未启用，导入器会自动降级为
每批 1,000 行的 `REPLACE`，功能不受影响，但全量导入会更慢。

正式全量演练前，应在受控环境评估并启用 `local_infile`。客户端只允许 MySQL 读取
导入器刚生成的指定临时分片，拒绝其他路径。

## 安全约束

- 导入器不在 Web 服务启动时自动运行；
- 不复权是当前唯一权威价格事实层；
- 缺失值写入 `NULL`，不得用零替代；
- 当前只写 v2 影子表，不切换线上 Repository；
- 全量发布前必须核对 17,036,064 行基线；
- 临时 TSV 成功提交或失败回滚后都会删除。
