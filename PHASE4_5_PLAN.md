# 量化回测 4.5 阶段目标与验收

## 阶段定位

第四阶段的参数研究与策略稳健性分析已经完成。4.5 阶段以功能完善、数据准确性和工程易用性为核心，在不破坏现有行情分析、策略回测、定投回测及实验结果的前提下，补齐区间分析能力，修复定投指标口径，统一前后端启动流程，并将浏览器本地数据迁移到 MySQL 服务。

## 阶段目标

1. 在行情分析页面增加区间涨跌幅计算和展示能力；
2. 修复定投收益页面的数据计算错误，统一最大回撤等指标口径；
3. 优化启动脚本，实现前端与后端服务同步启动；
4. 将现有 IndexedDB 数据全面迁移到 MySQL，由服务端统一提供持久化能力；
5. 保证历史数据可迁移、核心功能无回归、异常状态可定位。

## 功能范围

### 1. 行情区间涨跌幅

- 在行情分析页面支持选择起止日期或图表区间；
- 以区间首个有效交易日的收盘价为基准，以末个有效交易日的收盘价为结束值；
- 按 `(结束收盘价 - 起始收盘价) / 起始收盘价 × 100%` 计算区间涨跌幅；
- 同时展示实际起止交易日、起止收盘价、涨跌金额和涨跌幅；
- 非交易日边界自动映射到区间内最近的有效交易日，并在界面中展示实际采用日期；
- 数据不足、起始价格无效或区间顺序错误时，不显示误导性结果，并给出明确提示。

### 2. 定投收益指标修复

- 全面核对定投页面的累计投入、当前市值、收益金额、收益率和最大回撤；
- 定投收益率以累计投入资金为成本基础，不将后续外部入金误判为投资收益；
- 最大回撤基于剔除现金流影响后的单位净值序列计算，避免新增投入掩盖真实回撤；
- 页面摘要、历史结果、结果对比和导出数据使用同一套指标计算函数；
- 对零投入、仅一期投入、无有效行情和净值不可计算等边界情况返回明确状态；
- 为现金流、净值和最大回撤补充可人工核对的测试样例。

### 3. 前后端同步启动

- 调整根目录 `start.bat`，一次启动前端开发服务和后端服务；
- 启动前检查根目录与 `server` 目录依赖，缺失时给出明确提示；
- 前后端使用独立进程，任一服务启动失败时保留可读错误信息；
- 启动日志明确标识前端地址、后端地址和运行状态；
- README 统一说明一键启动和前后端单独启动方式；
- 不在启动脚本中写入数据库密码或其他敏感配置。

### 4. MySQL 数据持久化

- 由服务端统一连接 MySQL，浏览器不直接访问数据库；
- 将行情数据集、K 线数据、可视化策略、策略版本、草稿、回测结果、权益曲线、参数实验和实验明细迁移到 MySQL；
- 前端 Repository 改为调用服务端 API，不再以 IndexedDB 作为业务数据的最终存储；
- 建立数据库 schema、版本迁移机制、必要索引和唯一约束；
- 大批量行情和实验明细采用分页或分批读写，避免一次性传输和渲染全部数据；
- 服务端对请求参数、数据归属关系和写入事务进行校验，错误返回稳定业务码；
- 数据库连接信息仅从服务端环境变量读取，并在 `.env.example` 中提供空值模板；
- MySQL 不可用时展示明确的服务状态，不静默回退到浏览器缓存造成数据分叉。

## 数据迁移与兼容

- 提供一次性 IndexedDB 导出与 MySQL 导入流程，覆盖当前数据库中的全部业务表；
- 导入前校验 schema 版本、主键、外键关系和数据 checksum；
- 重复导入必须幂等，不产生重复数据；
- 迁移完成后对比各表记录数，并抽样核对行情、策略、回测和实验结果；
- 在用户确认迁移成功前不自动删除浏览器原数据；
- 迁移失败时保留失败原因和进度，允许修复后重试；
- 历史回测与实验快照保持不可变，迁移后仍能复现和查看；
- Excel、CSV 和 JSON 导入导出能力继续可用，其数据口径与 MySQL 中的记录一致。

## 实施顺序

- 增加行情区间涨跌幅；
- 统一定投净值、收益和最大回撤算法；
- 补充单元测试与历史结果回归测试。

### 阶段 2：MySQL 服务端基础

- 设计数据表、索引和迁移版本；
- 增加数据库配置、连接检查和事务封装；

### 阶段 3：前端数据层迁移

- 将现有 Dexie Repository 替换为服务端 API Repository；

- 完成 IndexedDB 数据迁移工具和迁移校验。

### 阶段 4：启动流程与验收

- 更新一键启动脚本和环境配置示例；
- 更新 README；
- 完成全量回归、构建、迁移和异常恢复测试。

## 测试计划

### 单元测试

- 区间涨跌幅的上涨、下跌、持平和非交易日边界；
- 定投现金流归一化净值和最大回撤；
- 零投入、单期投入、价格无效和数据不足场景；
- API 参数校验、数据转换和错误码映射；
- 数据库迁移脚本的升级、重复执行和回滚保护。

### 集成与回归

- 行情、策略、回测、定投和实验数据可以完整写入并从 MySQL 读取；
- IndexedDB 迁移前后记录数、关键字段和结果指标一致；
- 页面刷新或更换浏览器后，可以通过服务端读取既有数据；
- 数据库断开、接口超时和部分写入失败时，页面提示清晰且不产生脏数据；
- 原有行情分析、策略工作室、回测和参数实验功能无回归；
- 前端测试、类型检查、生产构建和服务端类型检查通过。

## 验收标准

- [ ] 用户可以选择任意有效行情区间并查看准确的区间涨跌幅；
- [ ] 区间边界落在非交易日时，界面展示实际采用的交易日；
- [ ] 定投累计投入、收益金额、收益率和最大回撤通过人工样例核对；
- [ ] 新增投入不会被计为收益，也不会掩盖单位净值回撤；
- [ ] 根目录启动脚本可以同步启动前端和后端服务；
- [ ] 全部业务数据由 MySQL 持久化，前端不再依赖 IndexedDB 保存最终数据；
- [ ] 现有 IndexedDB 数据可完整、幂等地迁移到 MySQL；
- [ ] 数据库不可用时有明确提示，不产生本地与服务端数据分叉；
- [ ] 敏感配置不进入前端构建产物、日志或 Git 历史；
- [ ] 自动化测试、前后端类型检查和生产构建通过；
- [ ] README 更新环境配置、启动方式、迁移流程和故障排查说明。
