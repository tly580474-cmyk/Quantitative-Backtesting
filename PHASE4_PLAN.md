# 量化回测第四阶段：参数研究与策略稳健性分析

## 1. 阶段定位

第四阶段在现有单标的策略回测、定投回测和可视化策略工作室之上，建设可重复的“策略实验室”。用户可以为策略参数定义搜索空间，批量运行回测，对训练区间与验证区间进行对比，并通过参数敏感性和滚动验证判断策略是否过度拟合。

核心闭环：

```text
选择数据集与策略
        ↓
定义参数空间与评价指标
        ↓
批量回测任务调度
        ↓
排行榜 / 热力图 / 参数敏感性
        ↓
样本外验证 / Walk-forward
        ↓
保存实验 → 应用候选参数 → 单次复核回测
```

第四阶段优先解决“一个策略应该使用什么参数，以及这些参数是否稳健”，暂不扩展多标的组合和实盘交易。

## 2. 阶段目标

1. 支持内置策略和可视化策略的参数批量实验；
2. 支持网格搜索和可复现随机搜索；
3. 支持训练集、验证集和 Walk-forward 滚动验证；
4. 提供排行榜、收益/风险散点图、二维参数热力图和敏感性分析；
5. 形成兼顾收益、风险、交易样本量和样本外表现的稳健性评价；
6. 实验可以暂停、取消、恢复、保存和导出；
7. 完成现有前端性能、响应式和无障碍质量收口。

## 3. 范围与优先级

### 3.1 P0：工程与口径基线

- 统一单次回测与批量回测的纯计算入口，避免维护两套撮合逻辑；
- 为回测配置、策略参数、数据 checksum 和引擎版本生成实验指纹；
- 引入确定性随机种子，确保随机搜索结果可复现；
- 明确日期切分规则，训练集和验证集按交易日切分且不得重叠；
- 主页面、策略工作室和图表按需加载，消除当前主包体积警告；
- 完成 375px、768px、1024px、1440px 响应式检查；
- 补齐键盘焦点、图标按钮标签、颜色之外的状态表达和 `prefers-reduced-motion`。

### 3.2 P1：参数实验 MVP

- 新增“策略实验室”页面；
- 选择一个本地数据集、一个内置策略或可视化策略版本；
- 数值参数支持最小值、最大值、步长和候选值列表；
- 枚举参数支持多选候选值；
- 自动计算组合数量并在运行前提示预计任务量；
- 网格搜索设默认硬上限，超限时要求缩小范围或改用随机搜索；
- 随机搜索支持样本数量和随机种子；
- 批量任务进入 Worker 执行，支持进度、预计剩余量、取消和失败重试；
- 单个组合失败不应中止整个实验，失败原因单独记录；
- 支持按累计收益、年化收益、夏普比率、最大回撤、Calmar 比率和稳健得分排序；
- 从排行榜一键打开完整回测详情，或将参数应用回策略回测页面。

### 3.3 P1：样本外验证

- 支持按比例或指定日期切分训练集和验证集；
- 参数仅在训练集排名，验证集只用于复核，不参与选优；
- 同时展示训练与验证指标及其衰减幅度；
- 验证区间数据不足、无交易或指标不可计算时给出明确状态；
- 支持“先按训练集取 Top N，再运行验证集”，控制计算量；
- 保存切分方式和实际交易日起止日期，保证实验可复现。

### 3.4 P2：Walk-forward 与稳健性分析

- 支持固定窗口和扩展窗口两种 Walk-forward 模式；
- 配置训练窗口、验证窗口和滚动步长；
- 每个窗口独立选参，禁止把后续窗口信息泄漏到前序窗口；
- 拼接各验证窗口权益曲线，形成样本外综合结果；
- 展示各窗口入选参数的漂移情况；
- 二参数实验提供热力图，其他参数固定或按筛选条件聚合；
- 提供参数邻域稳定性，识别“孤立尖峰”参数；
- 提供收益与最大回撤、收益与交易次数的散点图；
- 稳健得分必须展示组成项，不以单一黑箱分数替代原始指标。

### 3.5 P2：实验管理与数据交换

- 实验草稿和结果保存到 IndexedDB；
- 支持实验重命名、复制、删除和按创建时间/得分排序；
- 未完成实验保存进度，刷新页面后可从剩余组合继续；
- 恢复前校验数据 checksum、策略版本和引擎版本；
- 支持导出实验摘要 CSV 和完整实验 JSON；
- 数据库 Excel 导出增加实验概要表，但大规模组合明细继续使用 JSON/CSV，避免工作簿失控；
- 支持导入兼容版本的实验定义，不直接信任外部计算结果。

## 4. 明确不纳入第四阶段

- 多标的组合回测、资产配置和跨品种信号；
- 遗传算法、贝叶斯优化和模型自动循环改写策略；
- 分钟、Tick 和实时行情；
- 云端任务队列、多人协作和策略市场；
- 实盘下单、券商接口和自动交易；
- 将用户完整行情或批量实验结果上传给 AI 服务；
- 以验证集或测试集反复调参后宣称为样本外结果。

以上能力可作为第五阶段候选，不应混入本阶段 MVP。

## 5. 关键业务口径

### 5.1 参数空间

- 整数周期参数必须保持整数；
- 浮点步长使用十进制定点展开，避免 JavaScript 浮点累计误差；
- 参数组合在进入 Worker 前完成去重和合法性校验；
- 可视化策略参数继续使用 DSL 中的参数声明和约束；
- 不允许优化没有声明为可调参数的 DSL 常量；
- 同一实验的参数顺序固定，保证组合 ID 稳定。

### 5.2 数据切分与防泄漏

- 指标预热可以读取训练段开始日前的历史数据，但绩效统计从训练段正式开始日计算；
- 验证段允许读取其开始日前的历史数据用于指标预热，不得将预热期收益计入验证结果；
- 选参只读取训练指标；
- Walk-forward 的第 N 个窗口只能使用该窗口训练结束日及以前的数据选参；
- 所有图表明确标识训练、验证和预热区间。

### 5.3 排名与稳健得分

默认排名不只追求最高收益。稳健得分建议归一化后组合：

```text
30% 样本外 Calmar
25% 样本外年化收益
20% 回撤控制
15% 训练到验证的表现保持率
10% 参数邻域稳定性
```

约束规则：

- 交易次数低于用户设定阈值时降权并显示“样本不足”；
- 无法计算的指标不能静默按 0 处理；
- 得分公式和权重保存在实验快照中；
- 用户可切换原始指标排序，稳健得分仅作为辅助；
- 界面持续展示“历史结果不代表未来表现”。

## 6. 技术方案

### 6.1 模块划分

```text
src/features/experiments/
  ExperimentLabPage.tsx       实验配置与运行入口
  ParameterSpaceEditor.tsx    参数空间编辑器
  ExperimentProgress.tsx      进度、取消与失败信息
  ExperimentLeaderboard.tsx   排行榜和筛选
  RobustnessDashboard.tsx     热力图、散点图和敏感性
  WalkForwardReport.tsx       滚动验证报告
  parameterSpace.ts           参数展开与随机采样
  scoring.ts                  排名和稳健得分
  split.ts                    训练/验证窗口生成
  scheduler.ts                分批调度、恢复和取消

src/db/
  experimentRepository.ts

src/workers/
  experiment.worker.ts
  experimentProtocol.ts
```

### 6.2 执行架构

```text
实验配置校验
    ↓
生成参数组合与窗口计划
    ↓
按批次发送给 Experiment Worker
    ↓
复用现有回测纯函数
    ↓
返回紧凑指标，按需保存权益曲线
    ↓
主线程增量更新排行榜和图表
```

实施约束：

- MVP 使用单 Worker 内批次执行，先测量后决定是否扩展 Worker 池；
- 默认仅保存全部组合的摘要指标，Top N 和用户收藏项保存完整交易及权益曲线；
- 每批完成后持久化 checkpoint，避免长任务因刷新全部丢失；
- UI 更新节流，不能每完成一个组合就触发整张表格重渲染；
- 大结果表使用分页或虚拟滚动；
- 图表组件和实验页面动态导入。

### 6.3 建议数据模型

```ts
interface StrategyExperiment {
  id: string;
  name: string;
  status: 'draft' | 'running' | 'paused' | 'completed' | 'cancelled' | 'failed';
  datasetId: string;
  datasetChecksum: string;
  strategyId: string;
  strategyVersion: string;
  visualStrategyVersion?: number;
  engineVersion: string;
  method: 'grid' | 'random';
  randomSeed?: string;
  parameterSpace: ExperimentParameter[];
  split: HoldoutSplit | WalkForwardSplit;
  ranking: RankingDefinition;
  totalRuns: number;
  completedRuns: number;
  createdAt: string;
  updatedAt: string;
}

interface ExperimentRunSummary {
  experimentId: string;
  combinationId: string;
  windowId: string;
  parameters: Record<string, number | string | boolean>;
  metrics: BacktestMetrics;
  robustScore?: number;
  status: 'completed' | 'failed';
  errorCode?: string;
}
```

数据库升级必须保留现有行情、策略和历史回测结果，并提供升级测试。

## 7. 页面与交互

桌面端采用三步式工作流：

1. 配置：数据、策略、参数空间、切分方式和排名指标；
2. 运行：组合数量、实时进度、取消、失败统计和资源提示；
3. 分析：排行榜、图表、样本外表现和候选参数操作。

交互要求：

- 运行按钮附近直接展示总组合数和预计回测次数；
- 高计算量操作需要二次确认，但不得用模糊的“可能较慢”代替具体数量；
- 所有异步按钮运行时禁用并显示状态；
- 图表颜色满足可辨识性，并提供表格作为数据替代；
- 盈利/亏损除颜色外同时使用正负号、标签或线型区分；
- 移动端不强塞热力图，多图表改为纵向卡片和可滚动数据表；
- 触控目标不小于 44×44px，正文移动端不小于 16px；
- 键盘 Tab 顺序与视觉顺序一致，焦点状态清晰可见。

视觉延续现有浅色、高信息密度金融分析界面。第四阶段不进行无关的整体换肤。

## 8. 实施顺序与工作量

### 阶段 0：质量基线与技术拆分（3～4 人日）

- 路由/页面动态加载和构建分包；
- 回测纯函数入口整理；
- 引擎版本、实验指纹和确定性随机工具；
- 响应式及无障碍基线修复。

### 阶段 1：参数空间与批量调度（5～7 人日）

- 参数空间编辑和组合生成；
- 网格/随机搜索；
- Worker 协议、批次调度、进度和取消；
- 单元测试与压力基准。

### 阶段 2：排行榜与实验持久化（4～5 人日）

- 指标摘要、排序和筛选；
- IndexedDB 实验表及 checkpoint；
- 实验管理、恢复和导出；
- 候选参数回填单次回测。

### 阶段 3：训练/验证与可视分析（5～6 人日）

- Holdout 切分和 Top N 验证；
- 训练/验证对照；
- 热力图、散点图和邻域敏感性；
- 稳健得分及可解释构成。

### 阶段 4：Walk-forward 与验收（5～7 人日）

- 固定/扩展窗口计划；
- 窗口独立选参和样本外权益拼接；
- 参数漂移报告；
- 性能、回归、移动端和无障碍验收。

预计总工作量：22～29 人日。建议先交付 P0 + P1，形成可用 MVP，再进入 P2。

## 9. 测试计划

### 9.1 单元测试

- 参数范围展开、浮点步长、去重和组合上限；
- 相同随机种子生成相同样本；
- 训练/验证和 Walk-forward 窗口边界；
- 预热数据不计入绩效；
- 排名、缺失指标、最低交易次数和稳健得分；
- 实验指纹和组合 ID 稳定；
- 取消信号和 checkpoint 恢复。

### 9.2 集成与回归

- 批量中的任一组合结果与相同配置的单次回测完全一致；
- 内置策略和可视化策略均可完成实验；
- Worker 中异常组合不会中断整个任务；
- 数据或策略版本变化后拒绝错误恢复；
- 数据库升级不影响旧行情、旧策略和旧回测结果；
- 导出后重新读取，实验定义和摘要一致；
- 现有 190 项测试继续通过。

### 9.3 性能与体验

- 以固定基准数据集记录 100、1,000、5,000 个组合的耗时和峰值内存；
- 1,000 个轻量组合运行期间主线程保持可交互；
- 取消请求在当前批次结束后及时生效；
- 10,000 条摘要的排序、筛选和滚动无明显卡顿；
- 首屏主包不再出现 500kB 以上的业务代码体积警告；
- 375px 页面无横向溢出；
- 仅使用键盘可以完成实验配置、启动和查看结果。

## 10. 验收标准

- [ ] 内置策略和可视化策略均可定义并运行参数实验；
- [ ] 网格搜索与固定种子的随机搜索结果可复现；
- [ ] 批量结果与同配置单次回测的核心指标一致；
- [ ] 支持进度、取消、失败隔离和 checkpoint 恢复；
- [ ] 训练集选优与验证集复核严格隔离；
- [ ] Walk-forward 不读取未来窗口数据；
- [ ] 排行榜、热力图、散点图和参数敏感性可用；
- [ ] 稳健得分展示全部组成项和降权原因；
- [ ] 实验可保存、复制、删除、恢复和导出；
- [ ] 前端测试、类型检查、生产构建和服务端类型检查通过；
- [ ] 主包体积告警消除，移动端和键盘操作验收通过；
- [ ] README 更新第四阶段功能、使用方式和风险说明。

## 12. 第四阶段后的候选方向

完成第四阶段后，再根据真实使用反馈在以下方向中选择第五阶段：

1. 多标的组合与资产配置；
2. 更高频率数据与交易日历；
3. 云端同步、任务执行和策略市场；
4. 仿真交易与券商只读/交易接口。

不建议在第四阶段尚未建立样本外验证能力前直接进入实盘链路。
