# 缠论结构接入行情分析工作区实施计划

## 0. 当前决策

本计划先建设“可复现、可解释、无未来函数”的缠论结构分析能力，再考虑买卖点、背驰、选股和策略回测。

首期接入位置为现有 `/analysis` 行情分析工作区，不直接改造个股详情页的简版价格走势图。个股详情页只保留“导入行情分析”入口，待主工作区稳定后再复用同一分析结果。

核心决策：

1. 缠论结构是独立的结构分析子系统，不作为普通技术指标注册到 `useIndicatorStore`。
2. 算法计算与图表绘制彻底分离；算法层不得依赖 React、Zustand 或 Lightweight Charts。
3. 首个正式版本固定算法口径，不在 MVP 阶段开放全部流派参数。
4. 所有结构同时记录端点时间与确认时间，候选结构不得作为已确认信号进入回测。
5. 日、周、月分别对对应周期 K 线独立计算，不把低周期笔直接拼成高周期笔。
6. 图表显示原始 K 线；去包含 K 线作为内部结构数据，可通过诊断模式查看。
7. 现有 HTML 测试图只作为交互与绘图参考，其中“测试中枢”不能直接视为正式中枢实现。

## 1. 背景与问题

当前行情分析工作区已经具备：

- 日、周、月 K 线聚合；
- 主图、成交量和独立指标副图；
- 技术指标配置侧栏；
- 筹码峰叠加；
- 区间选择与十字光标状态；
- 数据导入、复权切换、导出和回测信号标记。

缠论结构与普通指标存在本质差异：

- 分型、笔、线段、中枢是带拓扑关系的结构集合，不是单一数值序列；
- 最后分型、最后一笔和最后线段会随新 K 线变化；
- 结构端点出现时间通常早于结构确认时间；
- 算法口径变化会级联改变后续所有结构；
- 中枢需要绘制时间与价格二维矩形，不能只用 `LineSeries` 表达；
- 回测必须以确认时间而不是结构端点时间判断信号是否可用。

因此，本功能需要独立的数据模型、状态管理、绘制 Primitive 和防未来函数测试。

## 2. 目标

### 2.1 产品目标

1. 在行情分析主图叠加分型、笔、线段和中枢。
2. 明确区分已确认、候选和已失效结构。
3. 在当前 K 线和历史十字光标位置显示可解释的结构摘要。
4. 支持日、周、月周期独立分析。
5. 保存算法版本、算法配置、数据指纹与复权口径，使结果可以复现。
6. 为后续买卖点、背驰、多级别联立、选股和策略回测提供稳定结构协议。

### 2.2 工程目标

1. 建立纯 TypeScript 缠论计算引擎。
2. 使用 Lightweight Charts Primitive 批量绘制结构。
3. 保持现有指标、筹码峰、区间选择和回测标记正常工作。
4. 500～2,000 根日 K 的结构计算与绘制不产生可感知卡顿。
5. 建立逐 K 追加和完整重算的一致性验证。
6. 建立固定样本与黄金结果，阻止算法修改造成静默漂移。

## 3. 非目标

首期不包含：

- 自动交易和实盘下单；
- 以缠论名义直接输出确定性投资建议；
- 分钟、Tick 或盘口级实时缠论；
- 全市场批量缠论扫描；
- 用户自定义任意线段流派和中枢流派；
- 自动调参或根据历史收益选择缠论口径；
- 将当前简化“测试中枢”直接用于买卖点；
- 在算法与逐 K 验证完成前接入正式回测信号。

## 4. 首版算法口径

首版版本号建议为 `chan-v1`。

| 项目 | 首版口径 |
| --- | --- |
| 输入价格 | 默认使用当前行情分析数据的既定复权口径 |
| K 线展示 | 原始/复权后的普通 K 线 |
| 包含处理 | 按前序无包含 K 线方向合并 |
| 分型 | 严格顶底分型 |
| 笔 | 新笔定义 |
| 线段 | 标准特征序列，两类线段破坏判定 |
| 中枢 | 先实现笔中枢，再实现线段中枢 |
| 等值规则 | 包含允许等值；标准分型使用严格不等式 |
| 未完成结构 | 保留为 `candidate`，不得冒充确认结构 |
| 多周期 | 日、周、月独立计算 |

M0 阶段必须把以下歧义写成测试和配置常量：

- 第一组 K 线方向不明确时的包含处理；
- 相同最高价、最低价的时间点选择；
- 连续同类分型的极值保留规则；
- 顶底分型区间是否允许重叠；
- 新笔原始 K 线计数的起止边界；
- 特征序列包含处理的方向；
- 特征序列缺口的等值边界；
- 中枢接触一个价格点是否算重叠；
- 中枢延伸、结束、新生和扩展的边界；
- 未结束周 K、月 K 对结构状态的影响。

## 6. 数据协议

### 6.1 分析结果

```ts
interface ChanAnalysis {
  algorithmVersion: 'chan-v1';
  configHash: string;
  sourceFingerprint: string;
  period: 'day' | 'week' | 'month';
  adjustmentMode: 'none' | 'qfq' | 'hfq' | 'unknown';
  sourceBars: number;
  mergedBars: ChanBar[];
  fractals: ChanFractal[];
  pens: ChanPen[];
  segments: ChanSegment[];
  centers: ChanCenter[];
  current: ChanCurrentState;
  warnings: ChanWarning[];
}
```

### 6.2 结构公共字段

```ts
interface ChanStructureBase {
  id: string;
  startIndex: number;
  endIndex: number;
  startTime: string;
  endTime: string;
  status: 'candidate' | 'confirmed' | 'invalidated';
  confirmedAtIndex: number | null;
  confirmedAt: string | null;
}
```

约束：

1. `startTime/endTime` 描述结构位置。
2. `confirmedAt` 描述市场参与者最早何时可以知道该结构成立。
3. 任何交易信号只能引用 `confirmedAt` 不晚于信号日的结构。
4. ID 必须由算法版本、周期、端点和结构类型稳定生成，不能使用随机 UUID。
5. 所有结构保留原始 K 线索引和去包含 K 线索引，便于调试。

### 6.3 配置

```ts
interface ChanConfig {
  algorithmVersion: 'chan-v1';
  penMode: 'new';
  strictFractal: true;
  includeEqual: true;
  segmentMode: 'standard-feature-sequence';
  centerMode: 'standard';
}

interface ChanDisplayConfig {
  enabled: boolean;
  showFractals: boolean;
  showPens: boolean;
  showSegments: boolean;
  showPenCenters: boolean;
  showSegmentCenters: boolean;
  showCandidates: boolean;
  showDiagnostics: boolean;
}
```

算法配置与显示配置必须分离。关闭某一绘制层不能改变计算结果或 `configHash`。

### 7.1 顶部工具栏

新增一个“缠论”按钮：

- 未开启：普通按钮；
- 已开启：主色按钮并设置 `aria-pressed=true`；

不在顶部同时放置“分型、笔、线段、中枢”四个开关。

### 7.2 右侧检查器

```text
[指标] [缠论]
```

缠论页包含：

1. 当前结构摘要；
2. 显示层级开关；
3. 候选结构开关；
4. 算法版本与复权口径只读信息；
5. 诊断模式入口；
6. 风险提示：最后结构会随未来行情变化。

移动端复用现有 Drawer。

### 7.3 主图绘制规范

| 结构 | 默认样式 | 说明 |
| --- | --- | --- |
| 顶分型 | 向下小三角 | 高级层，默认可关闭 |
| 底分型 | 向上小三角 | 高级层，默认可关闭 |
| 已确认笔 | 金色实线 | 1.5～2px |
| 候选笔 | 橙色虚线 | 必须与确认笔有线型差异 |
| 已确认线段 | 蓝色实线 | 比笔略粗 |
| 候选线段 | 蓝色虚线 | 不使用仅颜色差异 |
| 笔中枢 | 紫色低透明矩形 | 显示上下沿 |
| 线段中枢 | 青色低透明矩形 | 与笔中枢可同时辨识 |

要求：

- 默认不显示结构编号，悬浮或诊断模式才显示；
- 中枢填充不得遮盖 K 线；
- 图表缩放、拖动和筹码峰切换时结构坐标同步；
- 图表可见范围变化只影响绘制裁剪，不影响结构计算；
- 键盘与屏幕阅读器可通过侧栏读取当前结构摘要。

### 7.4 十字光标

十字光标移动到历史 K 线时显示：

```text
所在笔：向上，28.42 → 33.15
所在段：向上线段
所在中枢：[29.80, 31.26]

```

普通查看模式展示“以完整数据最终划分后的历史归属”。后续增加回放模式时，必须切换为“当时已知结构”，两者不得混用。

## 8. 计算与性能策略

### 8.1 第一阶段

- 500～2,000 根 K 线采用前端纯函数 + `useMemo`；
- Primitive 始终挂载，显示切换只更新数据或可见性，不重建整张图；
- 结构计算不随十字光标和可见范围变化重复运行；
- 只对输入数据、周期或算法配置变化重新计算。

### 8.2 Worker 触发条件

满足任一条件后迁移到 `chanlun.worker.ts`：

- 2,000 根 K 线全量计算 P95 超过 50ms；
- 切换周期时主线程出现明显掉帧；
- 加入完整线段和多级别联立后单次计算超过性能预算；
- 后续分钟级数据进入同一工作区。

### 8.3 服务端触发条件

以下功能出现前不建设服务端缠论计算：

- 全市场结构扫描；
- 定时结构快照；
- Agent 批量读取结构摘要；
- 多用户共享同一结构缓存；
- 缠论因子研究与大规模回测。

## 9. 数据与周期治理

1. 行情分析结果必须记录复权口径；复权切换后强制重新计算。
2. 从个股详情导入分析时，日 K 默认上限由当前 320 根提升到至少 1,000 根。
3. 输入必须按时间升序、日期唯一且 OHLC 合法。
4. 停牌 K 线、零成交 K 线和不可交易标记不得被静默删除；处理策略写入警告。
5. 周/月 K 由完整日 K 聚合；当前未结束周期标记为未完成。
6. 未完成周/月 K 产生的最后结构只能是候选。
7. 更换数据起点可能改变最早一段结构，界面应显示有效预热范围。
8. 结构结果的 `sourceFingerprint` 至少覆盖时间、OHLC、周期和复权口径。

## 11. 测试计划

### 11.1 单元测试

- 包含与反向包含；
- 连续多 K 包含；
- 第一组方向不明确；
- 严格顶分型和底分型；
- 等高、等低不构成严格分型；
- 连续同类分型保留更极端端点；
- 新笔最小原始 K 线间距；
- 无效反向分型不结束当前笔；
- 特征序列包含与缺口；
- 两类线段破坏；
- 中枢三段重叠、点接触和无重叠；
- 中枢延伸、结束、新生和扩展；
- 未完成周期 K 的候选状态。

### 11.2 不变量测试

1. 输出结构端点索引始终在输入范围内。
2. 笔方向严格交替。
3. 线段方向严格交替。
4. 已确认结构必须具有 `confirmedAt`。
5. 候选结构不得生成已确认买卖点。
6. 同一输入和配置重复运行结果字节级一致。
7. 仅改变显示配置时分析结果不变。
8. 完整重算与逐 K 增量最终结果一致。

### 11.3 前缀与未来函数测试

对每个截点 `T` 执行：

```text
incremental(input[0..T]).confirmedStructures
==
full(input).structures.filter(confirmedAtIndex <= T)
```

如果完整历史导致早期候选端点被替换，替换不得反向改变当时已经触发的确认信号审计记录。

### 11.6 性能测试

| 数据规模 | 目标 |
| --- | --- |

- [ ] 复权口径、算法版本、配置哈希和数据指纹可查看。
- [ ] 缠论开关不影响现有指标、筹码峰、区间选择和回测信号。
- [ ] 中枢绘制使用正式算法，不复用测试 HTML 的简化结果。
- [ ] 前缀不变性和无未来函数测试通过。
- [ ] 500～2,000 根 K 线满足性能目标。
- [ ] 移动端与桌面端均可操作，工具栏无溢出。
- [ ] 用户可以从任一笔、线段或中枢追溯其构成元素与确认依据。

## 13. 风险与控制

| 风险 | 影响 | 控制措施 |
| --- | --- | --- |
| 原理论解释存在差异 | 不同软件结果不一致 | 固定 `chan-v1`，配置和黄金样本版本化 |
| 最后结构重绘 | 用户误判已经确认 | 候选虚线、状态标签、记录 `confirmedAt` |
| 线段算法复杂 | 边界错误级联到中枢 | M3 独立交付，开放特征序列诊断视图 |
| 数据起点不同 | 最早结构不同 | 显示预热范围，保存数据指纹 |
| 复权切换 | 历史极值和结构变化 | 复权口径进入缓存键并强制重算 |
| 图层过多 | K 线难以阅读 | 默认只开笔、中枢、候选；高级层放侧栏 |
| React 重渲染 | 十字光标卡顿 | Primitive 内绘制，结构计算与 hover 解耦 |
| 320 根行情不足 | 高级别结构不稳定 | 导入分析至少请求 1,000 根日 K |
| 未来函数 | 回测结论失真 | 增量引擎、确认时间、前缀不变性测试 |
| 算法改动静默漂移 | 历史结果不可复现 | 算法版本、配置哈希、黄金样本和快照测试 |

## 15. 推荐执行顺序

```text
M0 定义冻结
  → M1 去包含 / 分型 / 笔
  → M2 主图接入
  → M3 标准线段
  → M4 正式中枢
  → M5 时间点回放与防未来函数
  → M6 买卖点 / 背驰 / 多级别 / 筛选 / 回测
```

不得跳过 M0 直接从测试 HTML 复制算法，也不得在 M5 完成前把缠论结构用于正式回测结论。
