# 2026-07-19 市场观点智能体定时推送计划

## 目标

在现有市场资讯、实时行情和市场观点智能体之上增加定时邮件推送。服务随项目后端启动，不依赖浏览器页面常驻。

| 时间（Asia/Shanghai） | 报告 | 核心依据 |
| --- | --- | --- |
| 09:00 | 消息早报 | 隔夜及最近消息、可用盘面、开盘后的确认/证伪条件 |
| 12:00 | 财经午报 | 上午指数、涨跌家数、成交额、资金、热点板块及消息兑现情况 |
| 16:00 | 盘后总结 | 收盘指数与市场广度、量价、热点持续性、消息兑现和次日验证项 |

## 内容质量约束

1. 新闻证据仅使用官媒、专业财经、聚合三类来源，并合并同一事件的跨媒体重复报道。
2. 每条关键结论必须包含具体对象、方向或状态、证据以及可验证条件。
3. 禁止“保持关注”“谨慎乐观”“市场或有波动”等不影响判断的套话。
4. 盘面数字只能来自结构化行情数据；接口缺失时明确写出降级项，不允许模型补造。
5. 新闻观点使用 `[N1]` 引用，区分事实、综合推断和待验证信息。
6. 不承诺确定收益，不直接输出买卖指令，但必须给出可证伪的阈值和影响路径。

## 调度和可靠性

- 后端启动时注册 30 秒轮询调度器，仅在配置时点后的宽限窗口内触发。
- 默认仅工作日执行，避免周末生成缺乏盘面依据的午报和盘后报告。
- 使用数据库 `market_data_collector_runs` 记录运行键，保证同一日期、同一报告最多成功发送一次。
- 失败任务最多重试三次；内存状态和管理接口只暴露配置完整性、时点、收件人数和最近结果，不返回授权码。
- 邮件使用 SMTP SSL；密钥只存放在被 Git 忽略的 `server/.env`。

## 验收

- 三个时点和宽限窗口单元测试通过。
- 智能体提示词包含盘面事实、交叉验证、反证和下一时段验证清单。
- SMTP 配置缺失时不启动调度器，后端仍可正常运行。
- TypeScript 构建、服务端测试和根项目构建通过。
