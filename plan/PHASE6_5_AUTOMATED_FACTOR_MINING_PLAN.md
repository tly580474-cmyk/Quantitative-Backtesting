# 第六阶段扩展：遗传规划自动因子挖掘接入计划

## 1. 背景与结论

实验项目 `D:\github_public_repo\遗传算法自动因子挖掘` 已形成完整的遗传规划（GP）闭环，覆盖公式树生成、适应度计算、选择、交叉、变异、早停、去重、报告和候选因子持久化。现有自动化方向可行，但当前研究口径仍可能高估样本外表现，不应直接把实验项目的“通过因子”发布到本项目正式因子库。

本扩展的定位是建设“自动候选因子生成器”，而不是无人审批的因子上线系统。公式搜索、批量评估、恢复、报告和候选库可以自动化；锁定测试集评估、正式发布和策略使用必须保留门禁。

## 2. 当前审查基线

- 实验项目工程测试：`84 passed`。
- 公式树、算子白名单、检查点和因子库等工程结构基本完整。
- 当前输出只能证明流程跑通，不能直接证明真实样本外有效。
- 最新合成数据报告的训练集与测试集实际为同一面板，不能标记为真实 OOS。
- 默认将训练集和验证集合并参与选择，验证集随后又用于早停，验证集不独立。
- 测试集指标参与候选排序、去重和验收，存在测试集多重检验污染。
- 5 日重叠前向收益被按日复利并使用 `sqrt(252)` 年化，夏普和回撤口径不可靠。
- 实验项目标签为当日收盘至未来收盘，本项目研究引擎采用下一交易日开盘进入，二者需要统一。

## 3. 目标与边界

### 3.1 目标

1. 修正实验项目的训练、验证、测试和交易评价方法学。
2. 复用本项目已发布的 DuckDB/Parquet 研究快照，避免维护第二套研究数据事实源。
3. 建立版本化的因子公式 AST 和白名单算子协议。
4. 让 GP 产出进入候选区，并由本项目统一研究引擎复算。
5. 建立候选冻结、锁定测试、风险复核和人工发布门禁。
6. 支持任务追踪、断点恢复、随机种子复现和完整研究血缘。

### 3.2 不在本期范围

- 自动发布因子到正式因子目录；
- 自动生成实盘订单或自动进入策略组合；
- 分钟级、Tick 或盘口因子；
- 未完成时点化的财务数据挖掘；
- 允许任意 Python、JavaScript 或 SQL 表达式执行。

## 4. 目标流程

```text
已发布研究快照
  → 训练集 GP 进化
  → 验证集选代、筛选和去重
  → 冻结候选及公式版本
  → 锁定测试集一次性评估
  → 本项目统一交易口径复算
  → 稳健性、风险和成本复核
  → 人工批准发布正式因子版本
```

测试集评估完成后，不允许针对同一锁定测试区间回头修改公式、阈值或超参数。需要继续研究时，必须创建新实验版本并向后滚动测试区间。

## 5. 分阶段实施

### M0：冻结审查基线与可复现实验

- 记录实验项目版本、Python 依赖、默认配置和随机种子。
- 将合成运行和真实运行明确分型，合成运行禁止产生 `OOS passed` 状态。
- 给每次运行记录数据快照、股票池、日期切分、标签、算子集、超参数和代码 checksum。
- 增加方法学回归测试，覆盖数据切分、标签错期、测试集隔离和持有期收益。

验收：相同快照、配置和随机种子可重现相同候选及关键指标。

### M1：修正样本外与回测口径

- GP 选择适应度只读取训练集，默认关闭训练/验证合并。
- 验证集只用于早停、超参数选择、候选排名和相关性去重。
- 候选冻结后才允许读取测试集；测试指标不得反向参与候选筛选。
- 实现真正的 expanding/rolling walk-forward，并为每折保留完整预热历史。
- 统一为本项目口径：T 日信号、T+1 开盘进入、持有期末退出。
- 对多日持有实现非重叠调仓或错位子组合净值，正确计算夏普、回撤和换手。
- 增加手续费、滑点、涨跌停、停牌及不可成交约束。
- 对大规模公式搜索增加多重检验控制，至少报告试验数量、Deflated Sharpe 或等价校正指标。

验收：测试集在候选冻结前不可访问；5 日收益不再被当作独立日收益复利。

### M2：统一数据与公式协议

- Python 挖掘器改为读取本项目已发布 Parquet 研究快照。
- 股票池、复权、ST、次新股、停牌、涨跌停和市场过滤统一由快照/研究配置提供。
- 定义版本化 JSON AST：终端、常数、算子、窗口、参数和类型均显式表达。
- AST 只允许白名单算子，并声明依赖字段、最大预热期、复杂度和方向。
- 在服务端实现 AST 到 DuckDB SQL 的通用编译器，逐步替代固定 `switch` 编译方式。
- 建立 Python 与 DuckDB 的算子一致性样例，验证 NaN、窗口边界、排名和中性化结果。

验收：同一 AST 在 Python 和 DuckDB 上抽样计算结果在约定容差内一致。

### M3：候选因子接入

- Python GP 保持为独立离线 worker/CLI，不嵌入 Node 进程执行。
- 新增自动挖掘任务及候选因子状态：`draft`、`frozen`、`tested`、`rejected`、`approved`。
- 候选只保存 AST、公式展示文本、运行血缘和摘要指标，不直接成为 active 因子。
- 冻结候选交由现有 `factorRunner` 统一复算 IC、Rank IC、分层收益和日度序列。
- 正式发布复用现有 `factor_definitions` / `factor_versions`，并记录候选来源及审批信息。
- 任务支持限时、限内存、取消、检查点恢复和失败重试。

验收：候选未经批准无法出现在正式可选因子目录，也无法被策略模块引用。

### M4：稳健性与发布门禁

- 至少使用多个随机种子重复搜索，报告公式出现频率和指标分布。
- 按牛熊阶段、市场、行业和市值分组报告稳定性。
- 校验行业、市值、流动性和已知风格暴露。
- 报告覆盖率、换手率、容量、交易成本敏感性和因子衰减。
- 与正式因子库进行截面相关性和边际增益检验。
- 发布审批展示完整公式、依赖、预热、方向、风险警告和锁定测试结果。

验收：只有满足硬门槛且通过人工审批的候选才能生成正式因子版本。

### M5：前端与日常自动化

- 因子研究页增加“自动挖掘”任务入口、资源预算和进度展示。
- 区分训练、验证、锁定测试指标，醒目标记 synthetic、in-sample 和 OOS。
- 展示进化轨迹、公式复杂度、相关因子、暴露和失败原因。
- 支持候选对比、冻结、拒绝和提交审批，不提供直接自动上线按钮。
- 数据快照更新后可创建定期挖掘任务，但不得自动重复打开既有锁定测试区间。

## 6. 硬性研究规则

1. 训练、验证、锁定测试严格按交易时间顺序切分。
2. 任何用于选择公式、代数、阈值或超参数的数据都不再称为测试集。
3. 测试集每个实验版本只评估一次；测试失败不能在同一区间调参重试。
4. 时间序列算子必须读取评价起点前预热历史，但预热样本不进入统计。
5. 因子值只能使用信号时点已经可知的数据。
6. 交易评价必须包含可成交价格、持有期、换手和成本，不使用重叠标签直接伪造净值。
7. 合成数据只验收程序正确性，不参与因子有效性结论。
8. 负向因子必须显式记录方向或进行公式标准化，不能混用绝对 IC 与有符号 ICIR。
9. 所有正式结论必须绑定快照、公式版本、股票池版本和评价配置。
10. 数据库密码和其他凭据只从环境变量或密钥配置读取，不进入仓库。

## 7. 建议验收指标

指标阈值必须由研究配置版本化，初始可保留 `|RankIC|`、ICIR、IC t 值、相关性和样本外衰减等指标，但补充以下条件：

- 使用锁定测试区间指标，而非候选筛选区间指标；
- 夏普、回撤来自可交易的组合净值；
- 成本后收益为正，并报告换手与容量；
- 多随机种子和多个市场阶段下方向基本一致；
- 对行业、市值和流动性暴露有明确解释；
- 与已发布因子相比存在可验证的边际信息增益；
- 报告候选搜索总数和多重检验调整结果。

## 8. 推荐实施顺序与决策门

实施时先隔离验证实验项目，完成 M0～M1 并用小规模真实快照复验；通过决策门后，
再将已审查的最小运行时合入主项目：

1. 选择固定股票池和固定历史快照；
2. 使用严格训练/验证/测试切分运行多个随机种子；
3. 冻结少量候选；
4. 用本项目交易口径执行一次锁定测试；
5. 人工审查报告和公式；
6. 若结果稳定，再进入 M2～M3 的正式接入。

进入正式接入的决策门：方法学测试全部通过，真实锁定测试没有异常高夏普、零回撤等口径信号，且至少有一个候选在成本后、分阶段和暴露复核中保持有效。
